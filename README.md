# Zorrotza Aire

PWA для просмотра официальных датчиков качества воздуха вокруг промышленной площадки Profersa в районе Zorrotza, Bilbao. Приложение полностью статическое и рассчитано на публикацию в GitHub Pages.

## Что есть в приложении

- станции в радиусе 12 км и расстояние до Profersa;
- интерактивная тёмная карта Leaflet с географической подложкой OpenStreetMap/CARTO;
- PM2.5, PM10, NO₂, SO₂, O₃, CO и другие доступные параметры;
- последние значения, 24/72-часовые графики и полнота ряда;
- сравнение до четырёх станций без интерполяции пропусков;
- European Air Quality Index и отдельные справочные ориентиры ВОЗ;
- прозрачные состояния: живой API, сохранённый официальный ответ, офлайн и демонстрационный режим;
- установка на iPhone/Android/desktop и офлайн-кэш оболочки приложения;
- подробная методика и контекст инцидентов Profersa 2024 года.

## Источник данных

Основной источник — [API качества воздуха Open Data Euskadi](https://opendata.euskadi.eus/api-air-quality/?api=air-quality). На стороне браузера запрашиваются:

- `GET /air-quality/stations`
- `GET /air-quality/measurements/hourly/stations/{station-id}/from/{date}/to/{date}`

Если API временно недоступен, приложение использует последний сохранённый официальный ответ. Если его ещё нет, включается явно подписанный демонстрационный режим: синтетические значения никогда не выдаются за реальные измерения.

## Локальный запуск

Service Worker не работает при открытии `index.html` как обычного файла, поэтому нужен любой локальный HTTP-сервер:

```powershell
python -m http.server 4173
```

Затем открыть `http://localhost:4173/`.

## Публикация в GitHub Pages

1. Создать пустой GitHub-репозиторий и загрузить в него содержимое этой папки.
2. В репозитории открыть **Settings → Pages**.
3. В разделе **Build and deployment → Source** выбрать **GitHub Actions**.
4. Отправить изменения в ветку `main`. Workflow `.github/workflows/pages.yml` опубликует сайт автоматически.

Приложение использует только относительные пути, поэтому работает как в корне домена, так и по адресу вида `https://user.github.io/repository/`.

## Важные ограничения

- Станция измеряет воздух в своей точке, а не выброс непосредственно на территории Profersa.
- Концентрация и направление ветра могут поддержать гипотезу, но не устанавливают источник загрязнения.
- Цвет последнего часа нельзя напрямую сравнивать со среднесуточной рекомендацией ВОЗ.
- «Нет данных» не означает нулевую концентрацию.
- Это информационный интерфейс, а не система экстренного оповещения или медицинская рекомендация.
